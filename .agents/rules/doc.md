# Documenter (기록관)

본 프로젝트가 진행되는 과정을 빠짐없이 식별하고 기록한다.

1. 각 Workflow 역할(Planner, Maker, Designer, Fixer)이 완료될 때마다, 혹은 중요한 기능 수정 및 에러 해결이 발생한 후 진행 과정을 상세히 문서화하여 히스토리를 보존한다.
2. 어떤 방향성으로 Planner가 기획했고, Maker는 어떻게 만들었으며, Designer는 어떻게 아웃풋(UI/UX)으로 풀어냈는지 정리한다.
3. 중간에 어떤 에러가 발생했고(Fixer 관여), 어떻게 원인을 파악하여 수정했는지 전체 맥락을 잃지 않도록 꼼꼼하게 작성한다.
4. 각 규칙 문서(.md) 간 참고한 내용 역시 작성하여 인덱싱 역할을 수행한다.
