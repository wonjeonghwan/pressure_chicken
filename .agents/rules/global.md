# Global Rule (전역 원칙)

AI 에이전트(개발팀)가 본 프로젝트를 진행함에 있어 **반드시 지켜야 하는 최우선 원칙**입니다.
이 원칙은 각 Workflow Agent(Planner, Designer, Maker, Fixer, Documenter)의 개별 룰보다 우선합니다.

### 1. 결정은 사용자가, 실행은 AI가 (Approval-First)
중요한 아키텍처 결정, 아예 새로운 라이브러리의 도입, 데이터베이스 구조의 대대적인 변경 등 프로젝트의 방향성을 가를 수 있는 분기점에서는 임의로 예상하여 작업을 진행하지 않습니다.
반드시 "현재 이런 상황이며, A안(장단점)과 B안(장단점)이 있습니다. 어떻게 할까요?"라고 명확하게 사용자에게 질문하고, **사용자의 확답(승인)이 있기 전에는 다음 코드를 작성하거나 변경하지 않습니다.**

### 2. 지속적인 맥락 동기화 및 로깅 (Continuous Logging)
작업 중간중간 `doc.md` 명세에 따라 "어디서부터 어디까지 개선했고, 무슨 에러가 나서 어떻게 원인을 파악해 우회/해결했다"는 로그를 강제로 남깁니다. 기능이 하나 완성되거나, 세션이 길어지거나, 다른 에이전트 역할로 스위칭될 때 맥락이 유실되는 것을 막기 위함입니다.

### 3. 명확한 역할과 대상 인식 (Context Boundary)
현재 수정하거나 논의하는 대상이 **"프로젝트 내 Product Agent (Script, Director, Audience, Editor, Publisher 등)"**의 코드/로직인지, 아니면 **"우리 개발 워크플로우(Planner, Maker, Designer 등)"**의 방향성이나 룰을 수정하는 것인지 항상 명시적으로 분리하여 인지하고 대화합니다. 애매할 경우 즉시 사용자에게 물어봅니다.

### 4. 보안 및 환경 유지 (Security & Housekeeping)
- **보안**: API Key, 개인정보 등 민감한 데이터가 절대 소스 코드나 Git에 포함되지 않도록 관리합니다. `.env` 파일을 활용하고 `.gitignore` 설정을 수시로 점검합니다.
- **최적화**: 중복된 기능을 하는 파일이 생기지 않도록 관리하고, 불필요한 파일은 과감히 정리하여 운영하기 가장 좋은 "싱글 포인트 담당" 구조를 유지합니다.
